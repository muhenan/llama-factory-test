# LoRA

## 业界使用场景

在业界，微调方式的选择通常取决于任务性质：

- **全量微调（Full Fine-Tuning）**：适用于通用基础模型的微调，比如在大规模数据上训练一个新的通用模型。需要充足的算力和数据，目标是全面改变模型的知识和能力。
- **LoRA 微调**：适用于专有领域的定制化需求，比如 Agent 能力注入、特定业务场景适配、垂直领域知识增强等。数据量相对少，显存要求低，迭代更快。

简单来说：改模型"骨子里"的东西用全量微调，给模型"加技能"或"调性格"用 LoRA。

---

## 原理

全量微调（Full Fine-Tuning）需要更新模型所有参数，对于一个 7B 模型，光存梯度就需要几十 GB 显存，代价极高。

LoRA（Low-Rank Adaptation）的思路是：**不动原始权重，在旁边插入两个小矩阵，只训练这两个小矩阵。**

### 数学表示

对于模型中某一层的权重矩阵 $W$（形状为 $d \times d$），正常微调是直接更新 $W$。

LoRA 的做法是冻结 $W$，引入一个增量 $\Delta W$，分解为两个低秩矩阵的乘积：

$$\Delta W = B \times A$$

- $A$ 的形状：$r \times d$（$r$ 远小于 $d$）
- $B$ 的形状：$d \times r$

前向传播变为：

$$h = Wx + BAx = (W + BA)x$$

训练时只更新 $A$ 和 $B$，$W$ 始终不变。

### 为什么叫"低秩"

$BA$ 的形状是 $d \times d$，看起来和 $W$ 一样大，但它是由两个"瘦"矩阵相乘得到的，其秩最大为 $r$。

秩（rank）代表矩阵能表达的独立方向数。$r$ 很小时，$BA$ 只能表达一个低维子空间内的变化，这正是 LoRA 的归纳偏置：**微调时模型需要学习的东西，本质上是低秩的。**

---

## 小例子：参数量对比

假设某一层权重矩阵 $W$ 形状为 $1024 \times 1024$，LoRA rank $r = 8$：

| | 形状 | 参数量 |
|---|---|---|
| 原始 $W$ | $1024 \times 1024$ | 1,048,576 |
| LoRA $A$ | $8 \times 1024$ | 8,192 |
| LoRA $B$ | $1024 \times 8$ | 8,192 |
| **LoRA 合计** | — | **16,384** |

LoRA 只用了原始参数量的 **1.56%**，却能对这一层的表达能力进行有效调整。

实际模型中会对多个层（通常是 attention 的 Q、K、V、O 矩阵）都加 LoRA，但总参数量依然远小于全量微调。

## 关键参数

### LoRA Rank（$r$）

决定低秩矩阵的秩，直接控制 LoRA 的参数量和表达能力。

- $r$ 越大 → 参数越多 → 表达能力越强 → 显存占用越高
- $r$ 越小 → 参数越少 → 正则化效果越强 → 适合数据量少的场景

常用值：`4`、`8`、`16`、`32`。
SFT 身份类任务（数据量小）通常 `r=8` 足够；复杂任务可以试 `r=16` 或 `r=32`。

### LoRA Alpha（$\alpha$）

缩放系数，控制 LoRA 输出对原始权重的影响强度。

实际缩放比例为 $\alpha / r$，前向传播变为：

$$h = Wx + \frac{\alpha}{r} \cdot BAx$$

**为什么要除以 $r$？**
当 $r$ 变大时，$BA$ 矩阵的数值尺度会随之变化。除以 $r$ 起到归一化的作用，让你在调整 $r$ 时不需要重新调 $\alpha$。

**常见设置：**
- $\alpha = r$：缩放比例 = 1，LoRA 输出不放大也不缩小
- $\alpha = 2r$：缩放比例 = 2，LoRA 影响更大，学习更激进

例：$r = 8$，$\alpha = 16$ → 缩放比例 = $16/8 = 2$

一般推荐 $\alpha$ 设为 $r$ 的 1～2 倍，不建议设得过大（会导致训练不稳定）。
